NoGS VPN Checker

What is it?

NoGS is a Chrome Extension I made at the end of the 2022. It allows you to block websites depending on whether or not you have a VPN turned on. You can list which sites you want blocked and how precise you want the VPN check to be. It came about when a friend of mine asked me if it was possible to block certain sites if you don't have a VPN turned on. After some research, I found that something like this didn't exist as far as I could tell, and I decided I might as well make it myself.

How?

When figuring out the implementation, I figured a Chrome Extension would function best. They are lightweight, well supported and make it easy to insert functionality into the web browser. It also gave me a good opportunity to learn how to make a Chrome Extension. There only needed to be two basic functions to have a working program, the ability to determine whether or not the user is using a VPN and the ability to block a website. Blocking a website is fairly trivial and there are a lot of resources about how to do so online, however figuring out if the user is connected to a VPN is a bit trickier.

For the most part, there is nothing within the browser that says “hey, I’m using a VPN!”. However, we can take advantage of some of the properties of a VPN to determine whether they are being used. A flagship feature of a VPN is their ability to change your remote IP Address, effectively masking your own. As long as the user provides the program with a “default” IP address to be compared against, we can simply compare their current IP to the default one in the program. If they are different, we can assume they are connected to a VPN. This works, but there are two big issues. First, if you have a portable setup (i.e. a laptop), moving anywhere outside of your home will cause the program to wrongly believe you are connected to a VPN. Likewise, for many home networks, the remote IP address is dynamic and can/will be changed by your ISP on a whim making it a farily unreliable data point. The solution to this is to take full advantage of the information available to the program. IP Address have a bunch of data attached to them. You can determine someone’s general location very easily if you have their IP address (which is part of the reason people like to mask them with VPNs in the first place). Using this location data, the program can more accuratly guess at when a VPN is being used. For example, instead of checking for differences in the specific IP address, you could check for the difference in the country of that IP Address. This way, you could move all within the United States, but the moment you turn on your VPN to switch a different country (a common use for streaming sites), the program will recognize the VPN. By giving options on how the program should determine the VPN status, you can tailor fit it to function in most setups.

I used my previous experience with HTML, CSS, and JavaScript to develop this, however this was my first-time using Bootstrap. I had always heard of it and knew it had something to do with styling but didn’t really know much else. After working with it and learning how to take advantage of it, I can say (as a person who is very challenged when it comes to the “prettiness” of a UI) Bootstrap really comes in handy. It worked perfectly for create a simple, professional, and consistent look for the extension. I’m very happy with the way it turned out and I am certainly going to use it for webapps in the future.

You can find out more specifics on how to use the extension and implemntation on the README on the GitHub page. For the most up to date versions of the extension, check out the GitHub repository.

Status

For the most part, NoGS is complete with no future updates planned. However, if something comes up or some great feature comes to mind, I may consider updating it more. NoGS is free to download from Chrome Web Store and the source code is available at the GitHub repository.
